[]{#BLOX_002eBText_003a-geometry-management}

#### 1.42.5 BLOX.BText: geometry management {#blox.btext-geometry-management .subsection}

[]{#index-child_003aheight_003a-2}

**child: child height: value**

Set the height of the given child to be 'value' pixels.

[]{#index-child_003aheightOffset_003a-2}

**child: child heightOffset: value**

Adjust the height of the given child to be given by 'value' more pixels.

[]{#index-child_003awidth_003a-2}

**child: child width: value**

Set the width of the given child to be 'value' pixels.

[]{#index-child_003awidthOffset_003a-2}

**child: child widthOffset: value**

Adjust the width of the given child to be given by 'value' more pixels.

[]{#index-child_003ax_003a-2}

**child: child x: value**

Never fail and do nothing, the children stay where the text ended at the
time each child was added in the widget

[]{#index-child_003axOffset_003a-2}

**child: child xOffset: value**

This method should not be called for instances of this class.

[]{#index-child_003ay_003a-2}

**child: child y: value**

Never fail and do nothing, the children stay where the text ended at the
time each child was added in the widget

[]{#index-child_003ayOffset_003a-2}

**child: child yOffset: value**

This method should not be called for instances of this class.

[]{#index-heightChild_003a-2}

**heightChild: child**

Answer the given child's height in pixels.

[]{#index-widthChild_003a-2}

**widthChild: child**

Answer the given child's width in pixels.

[]{#index-xChild_003a-2}

**xChild: child**

Answer the given child's top-left border's x coordinate. We always
answer 0 since the children actually move when the text widget scrolls

[]{#index-yChild_003a-2}

**yChild: child**

Answer the given child's top-left border's y coordinate. We always
answer 0 since the children actually move when the text widget scrolls
